# 회선교환 vs 패킷교환

---

## 회선교환 방식

- `회선교환`은 송수신 단말장치 사이에서 데이터를 전송할 때마다 **통신경로**를 설정하여 데이터를 교환하는 방식이다.
- 회선이 단절되기 전까지 데이터를 전송하지 않을 때에도 다른 단말장치들이 이 회선을 이용할 수 없으므로 회선의 이용률 측면에서 불리하나, 독점적으로 사용되기 때문에 주변의 트래픽이 많다고 회선이 끊어진다든지의 장애는 없다.
- 대표적인 예로 전화망이 있다.

<br /><br />

## 패킷교환 방식

- `패킷교환`이란 일정한 **데이터 블록인 패킷**을 교환기가 수신측 주소에 따라 적당한 통신경로를 선택하여 전송하는 교환 방식이다.
- 전송하고자 하는 정보를 일정한 크기의 데이터로 분할한 후, 송수신 주소인 헤더를 각각에 부가한 패킷단위로 전송한다.

<br /><br /><br /><br />

# IP / IP Packet

---

- `IP(Internet Protocol)`는 **네트워크 계층**에 해당된다.
- IP는 지정한 IP 주소에 `패킷(Packet)`이라는 통신 단위로 데이터를 전달한다. IP 패킷은 출발지 IP, 목적지 IP와 같은 정보가 포함되어 있다.
- 패킷 단위로 전송하면 노드들은 목적지 IP에 도달하기 위해 서로 데이터를 전달하며, 데이터를 전송받은 서버도 IP 패킷을 이용해 응답을 돌려준다.

<br /><br />

## IP 프로토콜 한계

### 비연결성

- 패킷을 받을 대상이 없거나 서비스 불능 상태여도 패킷 전송

<br />

### 비신뢰성

- 중간에 패킷이 사라질 수 있음
- 패킷의 순서를 보장할 수 없음

<br /><br /><br /><br />

# TCP / UDP

---

## TCP / IP

- `TCP(Transfer Control Protocol)`는 IP 프로토콜보다 더 높은 계층인 **전송 계층**에 해당하여 IP 프로토콜의 한계를 보완할 수 있으며, 신뢰성 있는 바이트 스트림 서비스를 제공한다.
  - **바이트 스트림 서비스**란 용량이 큰 데이터를 보내기 쉽게 TCP 세그먼트라고 불리는 단위 패킷으로 작게 분해하여 관리하는 것을 말한다.
  - `TCP 세그먼트`에는 IP 패킷의 출발지 IP와 목적지 IP 정보를 보완할 수 있는 출발지 PORT, 목적지 PORT, 전송 제어, 순서, 검증 정보 등이 포함되어 있다.

<br /><br />

## TCP 특징

### **연결 지향 - TCP 3 way handshake(가상 연결)**

![img](/assets/img/blog/220713-tcpip.png)

1. 클라이언트는 서버에 접속을 요청하는 SYN(Synchronize) 플래그를 보낸다.
2. 서버는 클라이언트에게 요청을 수락한다는 ACK(Acknowlegement)와 SYN 플래그를 발송한다.
3. 마지막으로 클라이언트가 서버에게 ACK를 보내면 이 이후로부터 연결이 성립되며 데이터를 전송할 수 있다.

만약 서버가 꺼져있다면 클라이언트가 SYN를 보내고 응답이 없기 때문에 데이터를 보내지 않는다. 현재에는 최적화가 이루어져 3번 ACK를 보낼 때 데이터를 함께 보내기도 한다.

<br />

### **데이터 전달 보증**

TCP는 데이터 전송이 성공적으로 이루어진다면 이에 대한 응답을 돌려준다 → IP 패킷의 `비연결성 보완`

<br />

### **순서 보장**

만약 패킷이 순서대로 도착하지 않는다면 TCP 세그먼트에 있는 정보를 토대로 다시 패킷 전송을 요청할 수 있다. → IP 패킷의 `비신뢰성(순서를 보장하지 않음) 보완`

<br />

### **신뢰할 수 있는 프로토콜**

TCP는 같은 계층에 속한 UDP에 비해 상대적으로 신뢰할 수 있는 프로토콜이다.

<br /><br />

## UDP 특징

`사용자 데이터그램 프로토콜(UDT, User Datagram Protocol)`은 IP 프로토콜에 PORT, 체크섬 필드 정보만 추가된 단순한 프로토콜이다.

- 하얀 도화지에 비유(기능이 거의 없음) - 커스터마이징 가능
- 비 연결지향 - TCP 3 way handshake X
- 데이터 전달 보증 X
- 순서 보장 X
- 데이터 전달 및 순서가보장되지 않지만, **단순하고 빠름**
- 신뢰성보다는 **연속성이 중요한 서비스(ex. 실시간 스트리밍)에 자주 사용**됨
