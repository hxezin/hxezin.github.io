# 풀이 방법

---

이진 탐색의 기본 문제로, mid의 제곱과 입력 값 num을 비교하며 포인터를 이동하며 값을 찾으면 되는 문제다. 그런데 주어진 예제를 입력하면 값이 다 맞는데 정작 답을 제출하면 틀렸다고 나왔다. 🤬

![img](/assets/img/blog/220612.png)

<br />

내가 놓친 조건이 뭔지 고민하다가, `입력 값인 N의 길이가 800자리가 넘지 않는다`는 것을 확인했다. JavaScript에서 Number 타입의 최대값은 2^53-1이다. 정확한 값을 알 수 없지만 어쨌든 800자를 넘는다는 가정하에 `BigInt`를 사용했다.

`BigInt`는 `연산에서 Number와 혼합해 사용할 수 없다.` 그래서 BigInt와 연산이 되는 숫자에 전부 n을 붙여 BigInt로 만들어줬다. 마지막 결과도 뒤에 n이 붙은 채로 출력돼서 문자열로 바꿔줬다.

```js
let input = require("fs").readFileSync("example.txt").toString();

function solution(input) {
  let num = BigInt(input);
  let start = BigInt(1);
  let end = BigInt(num);

  while (start <= end) {
    let mid = (start + end) / 2n;

    if (mid ** 2n > num) {
      end = mid - 1n;
    } else if (mid ** 2n < num) {
      start = mid + 1n;
    } else {
      return mid;
    }
  }
}
console.log(solution(input) + "");
```
